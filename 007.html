This reporting period marks a major milestone for the DRC Dashboard, with the implementation of a complete dual-mode architecture supporting both Manager and DRC (Technician) views. The backend services underwent a significant domain-driven refactoring, splitting the monolithic services.py into purpose-built modules for referrals, tissue, distribution, designation, and organization domains.

The DRC Dashboard now features comprehensive visualization components including RidgeCharts for process timeline analysis, RadarCharts for quality scoring, and SunburstCharts for hierarchical disposition breakdowns. An interactive Interval Analysis Panel enables technicians to compare time distributions between recovery events with statistical metrics (Mean, Median, StdDev).

On the CPR Dashboard front, the V2 API integration is now complete with live data flowing to all tabs. New features include a Country Yearly Distribution table for 4-year rolling international trends and comprehensive tissue details endpoints. The shared UI component library received substantial enhancements with centralized type definitions migrated to the @rmleb-iip/types package for improved reusability across applications.

DRC Dashboard

Mode Switching Architecture

A complete dual-mode system allowing users to switch between Manager and DRC (Technician) views. Each mode presents different tabs, metrics, and visualizations tailored to the user's role.

  • ModeProvider context wraps the application for global mode state
  • ModeSelector component in header with navigation to default tab on mode change
  • Dynamic route registration supporting direct navigation to any tab
  • Separate tab configurations: TABS_DRC_MODE and TABS_MANAGER_MODE

DRC Mode Visualization Components

A comprehensive suite of atomic chart and KPI components organized by tab, enabling modular composition and reusability.

  • Impact Tab: CasesRecoveredKPI, DispositionBreakdownChart, DomesticDistributionMap, GlobalDistributionMap, QualityScoreChart
  • Overview Tab: D2PKPI, PlacementRateKPI, RecoveriesKPI, QualityLeaderboard, RecoveriesLeaderboard
  • Performance Tab: RecoveryChart, TissuesRecoveredKPI, TransplantabilityKPI, UDRAIChart
  • CSS Grid layout strings centralized in constants/gridLayouts.ts

Process Timeline Analysis

Interactive RidgeChart visualization showing recovery event distributions with statistical analysis capabilities.

  • RidgeChart component with category click handlers and multi-selection support
  • IntervalAnalysisPanel displaying Mean, Median, and Standard Deviation for selected intervals
  • 2-event selection constraint for interval comparison
  • Ridge plot analysis logic extracted to utils/timelineUtils.ts

Backend Services

Domain-Driven Service Refactoring

The monolithic services.py (2,160 lines) has been decomposed into a modular package structure under app/services/.

  • base.py (227 lines): Shared logic and helper functions
  • referral.py (566 lines): ReferralService with UDRAI interview time and technician leaderboard metrics
  • tissue.py (446 lines): TissueService with quality score and recovery time metrics
  • distribution.py (504 lines): DistributionService for geographic analytics
  • designation.py (86 lines): DesignationService for donor designation logic
  • organization.py (159 lines): OrganizationService for cascading filter options
  • dashboard.py (228 lines): Unified DashboardService facade

DRC Metrics API Endpoints

New DrcService and router providing DRC-specific metrics for technician performance analysis.

  • Quality score timeline (RecoveryTech, Descemet, Epithelium scores)
  • D2P (Death to Preservation) metrics
  • UDRAI interview completion times
  • Technician performance leaderboard
  • New database models: EyeDonor and MedSoc

Stack total label improvements and new padding options.
