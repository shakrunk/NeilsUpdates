üìã Executive Summary

This week marked a significant milestone for the RMLEB Internal Information Portal with the successful integration of live backend data into the CPR Dashboard. The Eye Donor analytics tab now pulls real-time metrics directly from our internal database, eliminating the dependency on static placeholder data and paving the way for evidence-based decision-making by the Community & Professional Relations team.

Key architectural improvements include a complete refactoring of the API client package to support dashboard-specific data fetching, enhanced filtering capabilities that allow users to query by date ranges, categories (Coroner, Hospital, Hospice), and institutions, and the introduction of visual indicators for mocked versus live data‚Äîensuring transparency during the transition period.

The DRC Dashboard received foundational restructuring with new Organization and Individual Statistics tabs, setting the stage for upcoming technician performance feedback features. Infrastructure improvements across CI/CD pipelines and dependency management ensure the platform remains maintainable and aligned with modern development practices.

üìä CPR Dashboard Enhancements

Eye Donor Tab ‚Äî Live Data Integration

The Eye Donors tab has been fully connected to the backend API, transitioning from placeholder visuals to actionable real-time analytics. This integration enables the CPR team to make informed decisions based on current organizational performance.

  • Created useEyeDonorData hook to manage fetching, state, and data transformation for KPIs and charts
  • Implemented donorService interface to communicate with /api/v1/cpr endpoints
  • Added mocked data status propagation through the data pipeline with isMocked prop support
  • Normalized age distribution charts to display percentage-based recovery rates instead of absolute counts
  • Reversed display order of age groups for improved visualization flow

Global Filter Implementation

A comprehensive filtering system now spans the entire dashboard, allowing users to slice data by multiple dimensions simultaneously.

  • Replaced YearMonthFilter with DateRangeFilter for flexible date selection
  • Created CategoryFilter with real business domain values: Coroner, Hospital, Hospice, etc.
  • Implemented FilterProvider and FilterContext for global state management
  • Memoized apiFilters to ensure referential stability and prevent unnecessary re-fetching

Architecture Improvements

  • Migrated from local donorService to shared CPRClient instance from @rmleb-iip/api-client
  • Reorganized pages directory to tabs for improved semantic clarity
  • Defined explicit TypeScript interfaces (AgeChartData, MonthlyChartData) to eliminate any usage
  • Removed deprecated useCPRData hook and unused service files

‚öôÔ∏è Backend API Development

Advanced Filtering Infrastructure

The backend now supports comprehensive filtering capabilities through a redesigned schema architecture.

  • Replaced YearMonthFilter with DashboardFilters schema supporting date ranges, categories, and institutions
  • Updated donor service SQL queries to include BASE_JOINS for Organization and Relationship tables
  • Refactored build_date_filter_clause to build_query_filter_clause for BETWEEN and IN clause support
  • Added type hints to query filter parameters for improved type safety

Donor Analytics Enhancements

  • Refined age distribution from 20-year to 10-year intervals (0-9, 10-19, ... 90+) for granular demographic analysis
  • Split AgeDistribution schema from single count to recovered_count and unrecovered_count
  • Changed recovery determination logic from Referral Status string to EyeTissueRecovered table join verification
  • Added RECOVERY_JOIN with T-SQL logic to handle ID format discrepancies

Developer Experience

  • Created run.py script to auto-open browser to API documentation on server start
  • Upgraded FastAPI from 0.119.0 to 0.122.0 with pinned Starlette compatibility
  • Added 'test' environment to allowed ENVIRONMENT literal in settings
  • Standardized assertion syntax across donor_service for improved readability

üîå API Client Package

Architecture Overhaul

The API client package underwent a fundamental restructuring to better serve dashboard-specific requirements.

  • Replaced generic CRUD operations with specialized read-only metric fetching methods
  • Implemented domain-specific CPRClient methods: Donors, Transplants, Distribution, Eye Details, Institutions
  • Converted all CPR endpoints from GET to POST requests for JSON body payload support
  • Updated DashboardFilters interface: replaced year/month with date_range object

Technical Improvements

  • Migrated environment variables from process.env to import.meta.env for Vite compatibility
  • Fixed entry points to resolve to compiled .js files in dist/ directory
  • Improved interceptor test reliability with proper async assertion handling
  • Added package entry point at index.ts for cleaner imports
  • Reset DRCClient to placeholder state pending finalized backend specifications
